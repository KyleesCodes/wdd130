Web development basics: The internet protocall is called the TCP/IP, which stands for transmission control protocall, internet protocall. www stands for World Wide Web. http stands for hyper text transfer protocall. When a web user opens a website or app, a request is made to the web server that holds those files. The web user is referred to as the client (front end) while the server is referred to as the back end. URL stands for uniform resource locator. The URL matches up to an ip address with the DNS (domain name system). The server the sends the files back to the client. Developers often use hosting companies that already have servers to store the files. Different languages run on the front end and back end. HTML and CSS are front end languages. Python and PHP are back end languages. Try to make sites small and effectient so they can be run on various devices, also try to make them accessible to those with impairments. HTML basics: HTML stands for Hyper Text Markup Language, which is a code used to structure a web page and its content. HTML consists of a series of elements, which you use to enclose, or wrap, different parts of the content to make it appear a certain way, or act a certain way. The enclosing tags can make a word or image hyperlink to somewhere else, can italicize words, can make the font bigger or smaller, and so on. If we wanted a line to stand by itself, we could specify that it is a paragrah by enclosing it in paragraph tags:

cat is very grumpy

The first p wrapped in angle brackets is the opening tag, followed by the content, and finished with a closing tag. This is an element. Elements can also have attributes that look like:

My cat is very grumpy

Attributes contain extra information about the element that you don't want to appear in the actual content. Here, class is the attribute name and editor-note is the attribute value. The class attribute allows you to give the element a non-unique identifier that can be used to target it with style information and other things. Some attribute have no value, such as required. Attributes that set a value always have: 1. A space between it and the element name (or the previous attribute, if the element already has one or more attributes). 2. The attribute name followed by an equal sign. 3. The attribute value wrapped by opening and closing quotation marks. You can put elements inside other elements too- this is called nesting. If we wanted to state that our cat is very grumpy, we could wrap the word "very" in a element.

My cat is very grumpy.

You do however need to make sure that your elements are properly nested. The elements have to open and close correctly so that they are clearly inside or outside one another.
